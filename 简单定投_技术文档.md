# 简单定投 (Simple DCA) 技术文档

## 1. 项目概述

“简单定投”是一个纯前端的轻量级单页面应用（SPA），运行在本地浏览器中。该项目的主要目标是将指数基金的定投纪律转化为具体的操作建议。应用通过用户录入的资金状况、持仓数据以及预设的业务规则（如 PE 百分位策略、核心/卫星仓位管理、定投周期控制），自动计算出“买入”、“卖出”、“持有”或“暂停”的具体动作建议，并支持一键确认执行及历史撤回。

## 2. 技术栈

本项目遵循了极其轻量、无构建工具的设计理念：
*   **HTML5**: 使用语义化标签和 `<template>` 实现界面框架与组件克隆。
*   **CSS3**: 原生 CSS。采用 Flexbox 和 CSS Grid 实现响应式布局。
*   **JavaScript (ES6+)**: 原生 JS（Vanilla JS）。不依赖任何第三方框架。
*   **数据存储**: 使用浏览器原生 `localStorage` 进行本地状态持久化。

## 3. 文件结构

*   **`index.html`**: 页面入口。包含界面的骨架（侧边栏、资金设置、基金列表、操作建议等）以及用于动态渲染基金卡片的 `<template>`。
*   **`app.js`**: 应用的核心逻辑层。包含了状态树的定义、基于策略的操作建议算法、DOM 渲染函数以及第三方估值数据的抓取逻辑。
*   **`styles.css`**: 应用的样式文件。定义了视觉主题及响应式适配规则。

## 4. 数据与状态管理

应用将所有的状态收敛在一个名为 `state` 的全局对象中，通过 `localStorage` 同步。

### 核心状态 (`state`)
*   **`availableCash`**: 当前可用于投资的**总资金池**。
*   **`rebalanceMonth`**: 年度再平衡设定的月份。
*   **`rules`**: 业务规则集合（包含核心仓比例、分位阈值、定投周期 `dcaCycleDays`、定投步数 `dcaBatches` 等）。
*   **`funds`**: 用户持有的基金列表，包含持仓金额和 `lastActionDate`。
*   **`actionHistory`**: 操作历史记录栈，用于支持多次撤回。

### 渲染流与持久化
*   **实时存储**：任意状态变更都会触发 `saveState()` 保存到 `localStorage`。
*   **手动备份**：支持“导出/导入数据备份”功能。用户可以将当前状态导出为本地 `.json` 文件，用于跨设备迁移或防丢失备份。

## 5. 核心业务逻辑与策略算法

### 5.1 资金规划与目标
*   **总资金池** = 持仓总金额 + 可投资金额。
*   **核心/卫星仓目标**：根据配置比例计算，并在侧边栏同步显示“目标金额”与“已持仓金额”。所有金额自动四舍五入取整。

### 5.2 策略建议触发规则
*   **定投周期限制**：若 `fund.lastActionDate` 距今小于 `dcaCycleDays`，则强制显示“等待定投日”。
*   **买入信号**：由 PE 百分位触发（卫星仓 < `satelliteBuyBelow`；核心仓分段触发）。

### 5.3 智能分批与分配算法 (Rolling DCA)
应用采用“总资金池 + 动态分批”的算法，自动解决用户资金变动时的节奏难题：
1.  **计算本期预算**：`本期定投总预算 = 当前可用总资金 / 建议定投步数`。
    *   *逻辑说明*：如果用户资金从 1,200 变 12,000，每期投入会自动从 200 升至 2,000。
2.  **识别理想需求**：计算所有符合买入条件基金的“理想买入缺口”（Gap）。
3.  **最终额度确认**：本期建议投入总额 = `Math.min(本期定投总预算, 总理想缺口)`。
4.  **比例分配**：按照各个基金缺口占总缺口的比例，对上述总额进行切分。

### 5.4 确认执行与撤回
*   **`executeAction`**: 自动更新对应基金的持仓金额，记录 `lastActionDate`。系统不自动扣减 `availableCash`。
*   **`undoAction`**: 从 `actionHistory` 栈中弹出最近一次记录，恢复基金持仓及日期。

## 6. 维护指南

1.  **修改策略**: 定位 `app.js` 中的 `buildFundAdvice`。
2.  **精度控制**: 所有的买入/卖出金额必须经过 `Math.round()` 取整。
3.  **防丢失**: 建议用户在进行大规模持仓修改后，使用侧边栏底部的“导出数据备份”功能保存一份 JSON。
